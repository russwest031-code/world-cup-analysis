# World Cup Analysis

一个面向移动端展示的世界杯比赛预测与分析应用。项目目标不是做静态赛程页，而是呈现一个可上线、可自动更新、可解释的 AI coding 项目闭环。

线上地址：https://world-cup-analysis.onrender.com/

## 功能亮点

- 外部数据接入：每日从公开 World Cup JSON 数据源拉取全量赛程、队伍、比分状态。
- 预测模型刷新：基于球队排名、近期状态、攻防指标和泊松分布，生成胜平负概率、最可能比分、信心指数。
- 出线动机建模：根据2026世界杯规则计算小组积分、净胜球、剩余场次、求胜强度、平局价值和净胜球需求。
- 攻防风格画像：结合球队基础指标和本届已完场表现，评估进球能力、大胜倾向、零封倾向和比赛节奏。
- 可解释分析：详情页展示预测依据、因素权重、实力对比、比分分布、情景推演和风险因素。
- 自动化更新：GitHub Actions 每天北京时间 15:00 自动运行刷新脚本，提交最新 `data.js` 缓存。
- 自动化上线：Render 部署 GitHub 最新提交，手机可以直接访问线上站点。
- 降级保护：外部数据源不可用时，脚本会回退到本地缓存数据，避免线上页面中断。

## 数据流水线

```text
openfootball/worldcup.json
        ↓
scripts/update-predictions.mjs
        ↓
数据标准化 + 特征补全
        ↓
出线形势 + 动机因子 + 攻防风格
        ↓
概率模型 + 泊松比分分布
        ↓
data.js 静态缓存
        ↓
Render 线上站点
```

## 预测逻辑

模型不是黑盒输出，而是将多类因素拆开：

- 球队实力排名：作为基础强弱差异。
- 攻防指标：进攻、防守、中场、近期状态四维评分。
- 近期状态：近 5 场结果转化为状态分。
- 出线形势：小组前二直接晋级，8 个成绩最好的小组第三进入 32 强；模型会识别求胜、保平和净胜球压力。
- 攻防风格：识别主动压迫型、防守控制型、开放进攻型、均衡型，并参考已完场比赛的大胜/零封/失球表现。
- 泊松分布：根据预期进球生成比分概率矩阵。
- 信心指数：结合结果概率集中度和因素一致性计算。

## 外部信号设计

赔率和专业球评属于高价值数据，但稳定来源通常需要 API key 或付费服务。当前版本在数据结构中保留了 `marketSignals` 和 `expertSignals`，线上不伪造赔率或球评共识；后续可以接入 The Odds API、API-Football、Sportmonks 或自建球评聚合源。

## 自动刷新

`.github/workflows/daily-prediction-update.yml` 配置了定时任务：

```yaml
cron: "0 7 * * *"
```

对应北京时间每天 15:00。刷新后 GitHub Actions 会自动提交新的 `data.js`，Render 再部署最新版本。

## 可扩展方向

- 接入付费体育数据 API，补充实时首发、伤停、技术统计和赔率。
- 加入模型回测页面，对比预测概率与实际赛果。
- 增加球队详情页，展示阵容、近期赛程、历史交锋。
- 将预测模型拆成独立服务，支持多赛事复用。

## 本地运行

```bash
npm start
```

刷新预测数据：

```bash
npm run update:predictions
```
